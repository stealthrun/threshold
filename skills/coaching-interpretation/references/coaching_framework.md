# Coaching framework — the athlete this engine reads for

`threshold` coaches **one** athlete model. Every read is graded against it: a good read
engages this engine and these limiters; a generic read that could apply to any runner is
a failing read. This is the functional model only — the traits that change a coaching
verdict, not a personal-data dump.

## The engine: speed-dominant

A short-to-middle-distance neuromuscular profile carried up to the road distances.

- Strong leg speed, turnover, and running economy; high tolerance for high-intensity
  effort. Intensity is where this athlete is comfortable.
- **Not** a big aerobic engine. Aerobic ceiling (VO2 max work) is not the bottleneck —
  it is already a relative strength.
- Calibrated max heart rate is **192**. Near-maximal work sits in the high 180s; the
  signals use this to tell "the heart reached its ceiling" from "the legs tapped out
  first."

The standing implication: this athlete can *produce* hard efforts more readily than they
can *absorb* and *aerobically sustain* them. Speed flatters the data. The coaching job is
to read past it.

## The limiters: where a read should live

These are the real constraints. A read that ignores them is describing, not coaching.

1. **Aerobic durability** — the ability to hold a controlled aerobic effort for a long
   time without drifting up. This is the true bottleneck, not top-end power. So:
   - An easy run that creeps toward tempo is **aerobic debt, not a bonus** — it spends
     durability instead of building it.
   - A long run held controlled and on plan is the limiter being trained *correctly* —
     worth naming as the win it is, not glossed over as "just a long run."

2. **Weekly volume tolerance** — the body is still adapting to higher loads, with real
   fatigue around the top of the current weekly range (~80 km/week). So:
   - Volume is never confused with quality. A big week is a cost, not an achievement.
   - Fatigue often hides *under* good splits — legs flat while paces look fine. The read
     should trust the body signal over the clean numbers.
   - A pace missed on heavy legs is **fatigue, not lost fitness** — the honest read is
     reassurance plus a recovery adjustment, never "you've regressed."

3. **Achilles tendinopathy risk** — flares after high-speed work (spikes, stiff racers).
   This is a genuine injury risk at the top end of intensity. So:
   - An Achilles niggle on a fast session is a **red flag**, never noise.
   - A session cut short because the Achilles spoke up was the **right call** — back-off
     is success, not a failed workout. Praise the decision, then protect the tendon.

## How a session gets read against this model

The deterministic signals (`stimulus_quality`, `vs_plan`, `key_signal`) say *what
happened*. This framework says *what it means for this athlete*. The mapping:

| what the signals show | the read this athlete needs |
|---|---|
| VO2 pace faded, heart rate stayed sub-max | legs tapped out before the engine did — diminished stimulus, not a bad day; the limiter (durability) is the reason |
| easy/long run drifted above easy | aerobic debt against the real bottleneck, not a free bonus |
| long run controlled and on plan | the durability limiter trained right — name the win |
| pace missed, body honestly fatigued | fatigue from load, not lost fitness — reassure, adjust recovery |
| splits clean but legs flat | volume tolerance talking under good numbers — trust the body |
| session cut on an Achilles niggle | the correct, protective call — affirm it, then guard the tendon |
| hit the plan cleanly, body fine | say so plainly — **do not manufacture concern** |

## Reading the session against the block

The signals say what happened; the limiters say what it means for this athlete; the
**block** says whether that meaning is good news or a problem *right now*. The same session
is read differently depending on where the athlete is in the plan — this is the difference
between coaching and describing.

- **Base / build phase.** The job is accumulating volume and aerobic durability, and
  fatigue is expected. An under-target quality session deep in a heavy build is the *cost of
  the work*, not lost fitness — reassure, keep the durability work going, adjust recovery if
  needed. A clean session here is durability being banked. Speed sharpness is not the point
  yet, so don't fault a flat top end.

- **Sharpening / competition phase.** Volume drops and quality should feel *sharp*. Now an
  off session matters more: paces that should feel easy but don't, or a quality effort that
  falls flat, is a real flag worth heeding, not something to push through.

- **Taper / race week.** The job is shedding fatigue so the engine arrives fresh. Slightly
  flat legs early in a taper are normal and expected; the read should reassure and protect.
  But a session that still feels heavy *late* in a taper, or any Achilles signal this close
  to a race, is a genuine concern — protect, don't chase. Never prescribe added load in a
  taper to "make up" a miss.

The throughline: an identical under-target effort is *expected fatigue* in a build and a
*flag* in a taper. Read the block before deciding which.

## The discipline that ties it together

This athlete is an engineer: curious, reads physiology, responds to the *why*. Explain the
mechanism, don't just flag the outcome. But explanation is not invention — the most common
coaching-engine failure is manufacturing a fatigue story on a clean session to sound
insightful. When the session was good and the body is fine, the coaching read is to say
that, plainly, and stop. Restraint is part of the model.
